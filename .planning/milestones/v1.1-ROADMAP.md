# Milestone v1.1: Build Settings Window

**Status:** Planning
**Phases:** 6-7 (2 phases)
**Total Requirements:** 8

## Overview

This milestone migrates Cmdex from Wails v2 to Wails v3 alpha (for native multi-window support) and implements settings as a proper application window instead of a modal dialog.

## Phases

- [ ] **Phase 6: Wails v3 Migration** - Migrate from v2 to v3 alpha, verify bindings, update configs
- [ ] **Phase 7: Settings Window** - Open/close settings as separate window with persistence and sync

## Phase Details

### Phase 6: Wails v3 Migration

**Goal**: Cmdex runs on Wails v3 alpha with all existing functionality intact

**Depends on**: Nothing (first phase of milestone)

**Requirements**: WMIG-01, WMIG-02, WMIG-03

**Success Criteria** (what must be TRUE):
1. User can run `wails dev` and launch the app in development mode with v3
2. User can run `wails build` and produce a working production binary with v3
3. All existing Go→React bindings (App.ts methods) work without modification
4. All React components render and function identically to v2
5. Build configuration files (wails.json, package.json, go.mod) reference v3 packages

### Phase 7: Settings Window

**Goal**: Users can open settings as a separate window that persists and syncs with the main window

**Depends on**: Phase 6

**Requirements**: SWIN-01, SWIN-02, SWIN-03, SWIN-04, SWIN-05

**Success Criteria** (what must be TRUE):
1. User can open settings via a menu item, button, or keyboard shortcut and a new application window appears
2. User can close the settings window by clicking the X button or pressing Escape key
3. Settings window position (x, y) and size (width, height) are restored on next open
4. Changing theme, font, or density in settings window immediately reflects in the main window
5. All existing settings (theme selection, font choice, density mode) function identically in window-based UI

## Milestone Summary

### Key Decisions

- Wails v3 alpha chosen for native multi-window support (Wails v2 required platform-specific workarounds)
- Settings window is a separate webview sharing the same Go backend and SQLite store
- Settings sync uses Wails v3 event system (`settings-changed` event)
- Window persistence uses Wails v3 window state APIs

### Technical Notes

- Wails v3 provides `window.Open()` and `window.Close()` methods
- Window state persistence via `window.GetState()` / `window.SetState()`
- Cross-window communication via `Events.On()` / `Events.Emit()`
- Settings remain stored in SQLite via existing GetSettings/SetSettings
