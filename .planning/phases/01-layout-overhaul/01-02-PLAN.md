## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Sidebar Popover → App.tsx delete handler | User-initiated delete action; command/category IDs are generated UUIDs from the DB, not user-typed input |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Spoofing | Popover confirmation | accept | Popover is client-side UI guard only; actual delete requires DB round-trip in Go backend. No auth bypass possible from spoofing the UI. |
| T-02-02 | Information Disclosure | Popover displays command/category name | accept | Desktop app; data is local to the user's machine. No PII exposure to external parties. |
| T-02-03 | Elevation of Privilege | Direct delete call without confirmation | accept | Popover IS the confirmation; backend validates the ID exists before deleting. No privilege escalation vector. |
</threat_model>
